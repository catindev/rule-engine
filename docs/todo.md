## В работе / следующие шаги

### UI редактор правил

Отдельное приложение для аналитиков. Хранит артефакты в БД (MongoDB).
Собирает снэпшот через API и деплоит его на сервер через CI/CD без участия разработчика.

Компоненты:
- CRUD артефактов (правила, пайплайны, словари)
- Контракт — схема входного объекта для валидации полей в UI
- Роли: Admin, Editor, Reviewer
- Жизненный цикл ruleset: DRAFT → REVIEW → ACTIVE → ARCHIVED
- Diff между версиями при review
- Сборка и публикация снэпшота из UI

### Стабильные идентификаторы проверок (checkId)

Каждая проверка должна иметь стабильный `checkId` не зависящий от версии правил.
Используется оркестратором для ветвления логики и хранения результатов как артефактов аудита.

### Хранение результатов проверок

Сохранение результатов в БД с привязкой к `checkId` и версии ruleset.
Обеспечивает воспроизводимость и возможность анализа выполненных проверок.
